目前已完成工作以及出现的问题

  1. 卸载默认渲染器并安装新的的渲染器

npm uninstall hexo-renderer-marked --save
npm uninstall hexo-renderer-markdown-it --save

  1. 配置Hexo 根目录下的 _config.yml 文件

#Markdown-it 配置
markdown:
render:
html: true
xhtmlOut: false
breaks: true
linkify: true
typographer: true
quotes: ‘“”‘’’
plugins:
- markdown-it-abbr
- markdown-it-cjk-breaks
- markdown-it-deflist
- markdown-it-emoji # 📌 表情
- markdown-it-footnote
- markdown-it-ins # 📌 下划线 (inserted)
- markdown-it-mark # 📌 高亮 (marked)
- markdown-it-sub # 📌 下标 (H2O)
- markdown-it-sup # 📌 上标 (x2)
- markdown-it-checkbox
- markdown-it-imsize
- markdown-it-expandable
- markdown-it-katex
- name: markdown-it-container
options: success

  1. 安装相关插件(报错安装不完全)

npm install markdown-it-abbr markdown-it-cjk-breaks markdown-it-deflist markdown-it-emoji markdown-it-footnote markdown-it-ins markdown-it-mark markdown-it-sub markdown-it-sup markdown-it-checkbox markdown-it-imsize markdown-it-expandable markdown-it-container --save

  1. 渲染失败不能正常显示公式、上/下标、高亮等等，但是图片插入成功。

  2. 更换一个集成度更高的渲染器

npm uninstall hexo-renderer-markdown-it --save

npm install @reimujs/hexo-renderer-markdown-it-plus --save

  1. 在 _config.yml 文件中，用新渲染器的配置 markdown_it_plus 替换掉原来的 markdown 部分：

markdown_it_plus:
html: true
xhtmlOut: true
breaks: true
linkify: true
typographer: true
quotes: “”‘’
rawLaTeX: false # 保持默认的 false 即可
plugins:
- plugin:
name: “@vscode/markdown-it-katex”
enable: true

说明：这个方案开箱即用，默认就集成并开启了 @vscode/markdown-it-katex 等插件，可以省去很多手动配置和排错的麻烦。

  1. 还是渲染失败不能正常显示公式、上/下标、高亮等等，但是图片插入成功

  2. 目前思考是否是因为没有执行hexo c的原因导致没有清空缓存，但是如果执行了这个命令就可能会删掉public/img下的所有新加的图片导致原页面渲染失败加载不了图片，后面可以尝试把_config.butterfly.yml中涉及到的图片改为链接的形式，或者可以将文件先备份然后尝试执行hexo c命令再做下一步。
